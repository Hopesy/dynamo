# 2.Dyna Browser集成工具

# 2.1简介

该插件的作用是将Dynamo程序集成到Revit工具栏,配合Data-Shapes节点包制作为Dynamo程序制作UI，效果更佳。也可以添加Dynamo程序集路径后直接搜索使用。官方地址为http://prorubim.com/en/tools/dyno/

安装完成后，打开Revit即可在管理面板找到Dynamo Browser，点击即可启用

# 2.2使用方法

1.启动只有右键点击Settings将准备好的Dynamo文件路径添加

2.右键buttons,按照下方的格式一一对应写入，重启Revit后Dynamo程序合集就义插件形式出现在Revit工具栏

注意:Dyna1.0以上版本支持Revit2020及以上版本，低版本Revit需要安装Dynao0.9;如果手动升级Dynamo版本并且更改了Dynamo源文件路径，将导致Dynao无法使用;在Dynao环境中运行的dynamo程序，生成的构件可以避免消失。

# 3.Dynamo模型导入Revit

# 3.1 ImportInstance.ByGeometry

通过此方法导入的模型将会变成Revit的导入符号

# 3.2 DirectShape.ByGeometry

此节点可以输入导入后的族类别、材质和族名称，但是导入的模型是Mesh三角面，且无法进入族编辑器进行二次编辑

# 3.3 FamilyType.ByGeometry（推荐）

此方法需要输入族名称、族类别、族样板路径等等。导入的族可以正常编辑。此外，Spring.FamilyInstance.ByGeometry（spring节点包）和SP.FamilyInstance.ByGeometry（Synthesize节点包）也可以，用法效果一致。

注意：常用模板为公制常规模型和公制体量，导入后可以建立面墙或者幕墙系统；对于复杂形体例如双曲面，以上三个节点均很可能无法导入成功，建议使用Rhino或者Grasshopper及RhinoInside建立后再导入Revit；内置节点FamilyType.ByGeometry仅支持Solid，其他两个节点支持surface和Solid

# 3.4导出STL

先在Dynamo中将模型导出为STL格式（MESH三角面），然后导入Rhino或者Sketchup，最后导入Revit

# 6.RevitBatchProcessor

该插件的作用为，使自己的Python或Dynamo任务脚本对多个Revit文件进行全自动批处理。下载地址为https://github.com/bvn-architecture/RevitBatchProcessor。

安装完成后，开始菜单里可以直接看到。双击即可使用，注意支持的Revit版本。

所支持的python文件模板

Output()

# 7.Dynamo自动保存

我们在Dynamo中编写脚本时，往往会因为操作不当，导致Dynamo运行卡死，只能强行关闭软件，随之而来的是我们编写的脚本也会因为没保存而前功尽弃。其实不然，Dynamo中其实自带了自动保存的功能。

我们打开Dynamo工作界面，同时按住CTRL+Shift+↑,会出现以下界面：

我们可以看到自动保存文件目录是**C:\Users\zhouhl\AppData\Roaming\Dynamo\Dynamo Revit\backup**

与Dynamo设置相关的文件一般都在

注意：60000毫秒就是自动保存的时间，1是备份的数量，可以自己更改
